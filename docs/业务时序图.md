## 图3-1 学生档案变更申请—审核—生效流程（流程3-1）

```mermaid
sequenceDiagram
    autonumber
    participant 学生
    participant 前端
    participant 后端
    participant 数据库
    participant 教师或管理员
    participant 通知服务

    学生->>前端: 进入个人档案页面
    前端->>后端: 获取当前档案
    后端->>数据库: 查询学生档案
    数据库-->>后端: 返回档案数据
    后端-->>前端: 返回档案
    前端-->>学生: 展示档案信息

    学生->>前端: 填写变更内容并提交申请
    前端->>后端: 提交档案变更申请
    后端->>后端: 校验(仅允许一条待审核)
    后端->>数据库: 写入 student_profile_update_request(PENDING)
    数据库-->>后端: 写入成功
    后端->>通知服务: 通知审核人(教师/管理员)
    通知服务->>数据库: 写入 system_notification
    后端-->>前端: 申请已提交
    前端-->>学生: 提示等待审核

    教师或管理员->>前端: 进入待审批列表
    前端->>后端: 获取待审核申请列表
    后端->>数据库: 查询 PENDING 申请
    数据库-->>后端: 返回申请及变更内容
    后端-->>前端: 返回列表
    前端-->>教师或管理员: 展示申请详情

    教师或管理员->>前端: 选择通过或拒绝并填写意见
    前端->>后端: 提交审核结果(APPROVED/REJECTED)
    alt 审核通过
        后端->>后端: 开启事务
        后端->>数据库: 更新申请状态为 APPROVED
        后端->>数据库: 同步更新 student_profile 主表
        后端->>通知服务: 通知学生“审核通过”
        后端->>后端: 提交事务
    else 审核拒绝
        后端->>数据库: 更新申请状态为 REJECTED，记录原因
        后端->>通知服务: 通知学生“审核拒绝及原因”
    end
    后端-->>前端: 审核完成
    前端-->>教师或管理员: 提示操作成功

    学生->>前端: 查看档案申请记录/通知中心
    前端->>后端: 获取申请记录或未读通知
    后端->>数据库: 查询
    数据库-->>后端: 返回
    后端-->>前端: 返回结果与意见
    前端-->>学生: 展示审核结果与意见
```

## 图3-2 岗位发布—投递—筛选—面试—反馈流程（流程3-2）

```mermaid
sequenceDiagram
    autonumber
    participant 企业
    participant 学生
    participant 前端
    participant 后端
    participant 数据库
    participant 通知服务

    Note over 企业,数据库: 阶段一：岗位发布
    企业->>前端: 进入岗位管理，创建岗位
    企业->>前端: 填写岗位信息(名称、描述、地点、薪资、类型等)
    前端->>后端: 提交岗位
    后端->>数据库: 写入 job_posting(OPEN)
    数据库-->>后端: 成功
    后端-->>前端: 发布成功
    前端-->>企业: 岗位已发布

    Note over 企业,数据库: 阶段二：学生投递
    学生->>前端: 浏览岗位大厅，查看岗位详情
    前端->>后端: 获取岗位列表/详情
    后端->>数据库: 查询 job_posting
    数据库-->>后端: 返回
    后端-->>前端: 返回岗位
    前端-->>学生: 展示岗位
    学生->>前端: 选择简历并投递
    前端->>后端: 创建投递(JobApplication)
    后端->>数据库: 写入 job_application(SUBMITTED)
    后端->>通知服务: 通知企业“新投递”
    后端-->>前端: 投递成功
    前端-->>学生: 提示投递成功

    Note over 企业,数据库: 阶段三：企业筛选与面试安排
    企业->>前端: 进入投递管理，查看投递列表
    前端->>后端: 获取投递列表
    后端->>数据库: 查询 job_application
    数据库-->>后端: 返回
    后端-->>前端: 返回投递及学生信息
    前端-->>企业: 展示投递列表

    企业->>前端: 筛选：进入面试
    前端->>后端: 更新投递状态为 INTERVIEW
    后端->>数据库: 更新 job_application
    后端->>数据库: 创建 interview 记录(可选或后续步骤)
    后端->>通知服务: 通知学生“进入面试”
    后端-->>前端: 成功
    前端-->>企业: 更新成功

    企业->>前端: 安排面试(时间、地点/链接)
    前端->>后端: 创建/更新面试(scheduledTime, location, meetingLink)
    后端->>数据库: 写入/更新 interview(SCHEDULED)
    后端->>通知服务: 通知学生“面试安排”
    后端-->>前端: 成功
    前端-->>企业: 安排成功

    Note over 企业,数据库: 阶段四：面试完成与最终结果
    企业->>前端: 填写面试反馈，标记面试完成
    前端->>后端: 更新面试状态为 COMPLETED，提交 feedback
    后端->>数据库: 更新 interview
    后端->>通知服务: 通知学生“面试结果”
    后端-->>前端: 成功

    企业->>前端: 更新投递最终状态(OFFERED/REJECTED)
    前端->>后端: 更新投递状态
    后端->>数据库: 更新 job_application
    后端->>通知服务: 通知学生“录用/拒绝”
    后端-->>前端: 成功
    前端-->>企业: 更新成功

    学生->>前端: 查看我的面试/通知
    前端->>后端: 获取面试安排与投递状态
    后端->>数据库: 查询
    数据库-->>后端: 返回
    后端-->>前端: 返回
    前端-->>学生: 展示面试安排与录用结果
```

## 图3-3 就业指导记录—沉淀—统计分析流程（流程3-3）

```mermaid
sequenceDiagram
    autonumber
    participant 教师
    participant 学生
    participant 前端
    participant 后端
    participant 数据库
    participant 通知服务

    教师->>前端: 进入指导记录/学生列表
    前端->>后端: 获取所指导学生及就业情况
    后端->>数据库: 查询 teacher_guidance、student_profile、job_application 等
    数据库-->>后端: 返回
    后端-->>前端: 返回学生列表及档案、投递、面试概况
    前端-->>教师: 展示学生就业情况

    教师->>前端: 选择学生，新增指导记录(内容、建议)
    前端->>后端: 提交指导记录(TeacherGuidance)
    后端->>数据库: 写入 teacher_guidance(教师ID、学生ID、内容、时间)
    数据库-->>后端: 成功
    后端->>通知服务: 通知学生“新指导记录”
    后端-->>前端: 保存成功
    前端-->>教师: 提示记录已保存

    学生->>前端: 进入就业指导/指导记录
    前端->>后端: 获取指导记录列表
    后端->>数据库: 查询 teacher_guidance
    数据库-->>后端: 返回
    后端-->>前端: 返回记录
    前端-->>学生: 展示指导意见，可据此修改简历/投递

    教师->>前端: 查看统计(指导次数、覆盖率等)
    前端->>后端: 获取统计指标
    后端->>数据库: 聚合 teacher_guidance、job_application、interview 等
    数据库-->>后端: 返回
    后端-->>前端: 返回统计结果
    前端-->>教师: 展示统计图表/报表
```

## 图3-4 通知触达流程（支撑流程3-4）

```mermaid
sequenceDiagram
    autonumber
    participant 业务模块
    participant 通知服务
    participant 数据库
    participant 用户
    participant 前端

    Note over 业务模块,数据库: 事件触发：档案审批/投递状态/面试安排/指导记录等
    业务模块->>通知服务: 生成通知( userId, category, title, content )
    通知服务->>数据库: 写入 system_notification( readFlag=false )
    数据库-->>通知服务: 写入成功
    通知服务-->>业务模块: 通知已记录

    Note over 用户,数据库: 用户查看与已读
    用户->>前端: 点击消息图标/进入通知中心
    前端->>通知服务: 获取用户未读/全部通知
    通知服务->>数据库: 查询 system_notification( userId, 分页 )
    数据库-->>通知服务: 返回通知列表
    通知服务-->>前端: 返回列表
    前端-->>用户: 展示通知列表

    用户->>前端: 查看某条通知/点击“全部已读”
    前端->>通知服务: 标记已读( markAllNotificationsAsRead 或单条 )
    通知服务->>数据库: 更新 readFlag=true
    数据库-->>通知服务: 更新成功
    通知服务-->>前端: 成功
    前端-->>用户: 更新未读数与列表状态
```

## 图3-5 管理员治理流程（支撑流程3-5）

```mermaid
sequenceDiagram
    autonumber
    participant 管理员
    participant 前端
    participant 后端Admin
    participant 数据库
    participant 通知服务

    Note over 管理员,数据库: 用户治理
    管理员->>前端: 进入用户管理
    前端->>后端Admin: 获取用户列表(分页、角色、状态、关键词)
    后端Admin->>后端Admin: requireAdminRole
    后端Admin->>数据库: 查询 sys_user
    数据库-->>后端Admin: 返回
    后端Admin-->>前端: 返回用户列表
    前端-->>管理员: 展示用户

    管理员->>前端: 新增/编辑用户、调整角色、启用/禁用
    前端->>后端Admin: createOrUpdateUser / updateUserStatus / updateUserRole
    后端Admin->>数据库: 更新 sys_user
    数据库-->>后端Admin: 成功
    后端Admin-->>前端: 成功
    前端-->>管理员: 操作成功

    Note over 管理员,数据库: 资料审批(教师/企业)
    管理员->>前端: 进入资料审核
    前端->>后端Admin: 获取 profile_update_request 列表(角色、状态、分页)
    后端Admin->>数据库: 查询 profile_update_request
    数据库-->>后端Admin: 返回
    后端Admin-->>前端: 返回申请列表
    前端-->>管理员: 展示待审核/已审核列表

    管理员->>前端: 查看详情，通过或驳回并填写意见
    前端->>后端Admin: approveProfileUpdateRequest / rejectProfileUpdateRequest
    后端Admin->>后端Admin: 校验状态为 PENDING
    alt 通过
        后端Admin->>数据库: 解析 payload，更新 teacher/employer 表
        后端Admin->>数据库: 更新申请状态 APPROVED
        后端Admin->>通知服务: 通知对应用户“审核通过”
    else 驳回
        后端Admin->>数据库: 更新申请状态 REJECTED，记录原因
        后端Admin->>通知服务: 通知对应用户“审核驳回及原因”
    end
    后端Admin-->>前端: 成功
    前端-->>管理员: 操作成功

    Note over 管理员,数据库: 通知发布与公共资源
    管理员->>前端: 发布系统通知
    前端->>后端Admin: sendSystemNotification( 指定用户或全体 )
    后端Admin->>数据库: 批量写入 system_notification
    后端Admin-->>前端: 成功

    管理员->>前端: 上传/维护公共资源、政策等
    前端->>后端Admin: 调用公共模块接口
    后端Admin->>数据库: 写入 public_resource 等
    后端Admin-->>前端: 成功

    Note over 管理员,数据库: 全局统计
    管理员->>前端: 进入数据统计/总览
    前端->>后端Admin: getDashboardData
    后端Admin->>数据库: 聚合用户、岗位、申请、面试、通知等
    数据库-->>后端Admin: 返回
    后端Admin-->>前端: 返回统计结果
    前端-->>管理员: 展示统计图表与汇总
```

## 使用说明

1. **在线渲染**：复制上述任意一个 ` ```mermaid ` 代码块（不含“图3-x”标题），粘贴到 [Mermaid Live Editor](https://mermaid.live/) 中即可生成时序图，可导出 PNG/SVG。
2. **插入论文**：导出图片后，在 Word 中插入图片，图题格式示例：“图3-1 学生档案审批业务时序图”。
3. **分图保存**：建议每个流程单独复制到一个 Mermaid 文档中渲染并导出，便于按图号插入论文对应位置。
